Operator Categories Demonstrated

Static Observables

  • of() - Emits values synchronously
  • from() - Converts array/promise to observable

Creation Operators

  • interval() - Sequential numbers at intervals
  • timer() - Delayed emissions

Transformation

  • map() - Transform each value
  • pluck() - Extract object property
  • scan() - Accumulate with intermediate results
  • reduce() - Accumulate to single result

Filtering

  • filter() - Values passing predicate
  • take() - First n values
  • takeUntil() - Until notifier emits
  • takeWhile() - While condition true
  • skip() - Skip first n values
  • first() - First value
  • last() - Last value
  • distinctUntilChanged() - When value changes
  • debounceTime() - After pause in emissions
  • throttleTime() - Emit then ignore for duration

Combination

  • combineLatest() - Latest values from multiple observables
  • forkJoin() - Last values when all complete
  • concat() - Sequential concatenation
  • merge() - Concurrent merging
  • zip() - Corresponding values
  • race() - First to emit

Flattening

  • mergeMap() - Map and merge
  • concatMap() - Map and concatenate
  • switchMap() - Map and switch (cancel previous)
  • exhaustMap() - Map and exhaust (ignore while active)

Error Handling

  • catchError() - Catch and handle errors
  • retry() - Retry on failure
  • retryWhen() - Custom retry logic

Subjects

  • Subject - Multicast, no initial value
  • BehaviorSubject - Has initial value
  • ReplaySubject - Replays last n values
  • AsyncSubject - Last value when complete

Multicasting

  • share() - Share single subscription
  • shareReplay() - Share and replay values

Practical Use Cases

Search with Debouncing

Use debounceTime to wait for user to stop typing before making API calls.

// Search input stream
searchInput$.pipe(
  debounceTime(300),
  distinctUntilChanged(),
  switchMap(term => searchAPI(term))
).subscribe(results => displayResults(results));

Auto-save with Throttling

Use throttleTime to limit save frequency while user is actively editing.

// User input stream
userInput$.pipe(
  throttleTime(2000),
  switchMap(data => saveToServer(data))
).subscribe(() => showSaveIndicator());

Load Related Data

Use switchMap to cancel previous requests when new data is selected.

// User selection stream
selectedUser$.pipe(
  switchMap(userId => loadUserDetails(userId))
).subscribe(details => displayUserDetails(details));

Multiple API Calls

Use forkJoin to wait for multiple independent API calls to complete.

// Load dashboard data
forkJoin([
  loadUsers(),
  loadPosts(),
  loadComments()
]).subscribe(([users, posts, comments]) => renderDashboard(users, posts, comments));
